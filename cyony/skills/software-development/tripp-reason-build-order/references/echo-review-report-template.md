# Echo Warden Review Report Template

Standard Markdown structure for every Echo/Warden review report.
Generated by `buildReviewMarkdown()` in `packages/external-agents/src/fileBus.ts`.

## Template

```markdown
# Echo Warden Review

## Review Identity

- **reviewId:** <UUID>
- **createdAt:** <ISO 8601>
- **reviewerRole:** openclaw_echo
- **verdict:** <pass | pass_with_notes | revise | block | escalate>

## Source Result

- **packetId:** <task packet UUID>
- **resultId:** <result packet UUID>
- **runId:** <swarm/agent run UUID>

## Summary

<Human-readable 1-3 sentence summary of the review>

## Issues

<Bullet list of specific issues found, or "(none)">

## Boundary Findings

<Bullet list of architectural/safety boundary violations found, or "(none)">

## Doctrine Findings

<Bullet list of doctrine/methodology violations found, or "(none)">

## Safety Findings

<Bullet list of safety/security concerns found, or "(none)">

## Proposed Changes Review

For each proposed change in the source result:

- <path> — <changeType> — risk: <low|medium|high>
  - requiresApproval: <true|false>
  - Echo note: <advisory assessment>

## Approval Boundary

- ⚠️ This Echo review is **advisory only**.
- ⚠️ This review does **NOT** approve mutation.
- ⚠️ **Tripp.Reason ApprovalGate remains authoritative.**
- ⚠️ **Eddie remains the final approver.**

## Recommended Next Action

<Clear, actionable next step — or "(none)">
```

## Rules

- Every report MUST include the Approval Boundary section verbatim.
- block/escalate verdicts MUST have at least one Issue or Safety Finding.
- The source result fields (packetId, resultId, runId) MUST link back to their originating files.
- reviewerRole is ALWAYS `openclaw_echo` — not configurable.
- This template is for CLI-generated reviews only; future live Echo may extend it.
