# Communication Anti-Patterns (from 2026-06-02 session)

## Anti-Pattern: Vague Hedging When Asked Tool Questions

**User asks:** "what does TokenMunch do?"

**BAD response pattern:**
- Give 10 paragraphs about related concepts
- Hedge with "it's part of a broader ecosystem"
- List features without explaining the core purpose
- User has to ask 3 more times before getting a direct answer
- User says "why are you being vague" multiple times

**GOOD response pattern:**
- Load the relevant skill first (pitfall #0 — non-negotiable)
- Deliver the Quick Answer: "TokenMunch indexes code/docs/data into searchable databases so you query 2K tokens instead of loading 100K tokens. Three tools: jCodeMunch, jDocMunch, jDataMunch."
- 3 sentences. Factual. Done.
- Offer to go deeper if user wants detail.

## The Rule

When user asks "what does X do" — they want WHAT IT DOES, not a history lesson, ecosystem overview, or comparison table. Lead with the 3-sentence answer. Context follows only if asked.

## Signals You're Doing It Wrong

- User says "why are you being vague" — you're hedging
- User says "just give me the answer" — you're over-explaining
- User asks the same question twice — your first answer was unclear
- User says "you can't explain to me what it does" — you failed to be direct

## The Fix

Load the skill. Read the Quick Answer section. Deliver it verbatim if needed. Then ask "want more detail on any part?"
